## 1. 프로젝트 개요

본 프로젝트는 사회복지 강의 교안을 입력하면 LLM이 복습 및 시험 대비용 학습노트를 자동 생성하도록 프롬프트 시스템을 설계한 과제이다.

테스트 자료는 **장애인복지론 1주차 1교시 「장애의 개념과 범주」 교안**을 사용하였다.

주요 검증 내용은 다음과 같다.

- Gemini 3 Flash, Claude Haiku 4.5, GPT-5.5 비교
- 최종 모델 Claude Haiku 4.5 선정
- 사회복지 강의 교안 기반 학습노트 자동화 시스템 설계
- Few-shot 예시 3개 설계
- 환각 검증 질문 5개 이상 설계
- 10턴 이상 실제 대화 로그 기반 문맥 유지 및 환각 검증
- 조건 변경 시 문맥 유지 여부 확인
- 최종 시스템 프롬프트 개선안 도출

## 2. 업무 과업 정의

### 2-1. 선택한 업무 과업

**사회복지 강의 교안 기반 학습노트 자동 생성**

### 2-2. 문제 정의

사회복지사 2급 및 정신건강사회복지사 준비 학습자는 강의 교안을 바탕으로 핵심 개념, 시험 포인트, 현장 적용점을 정리해야 한다. 그러나 강의 교안은 개념, 연도, 통계, 제도 변화가 함께 제시되어 초보 학습자가 핵심 구조를 빠르게 파악하기 어렵다.

따라서 본 프로젝트는 사회복지 강의 교안을 입력하면 복습 및 시험 대비용 학습노트를 자동으로 생성하는 LLM 프롬프트 시스템을 설계하는 것을 목표로 한다.

### 2-3. 타깃 사용자

| 구분 | 내용 |
|---|---|
| 주요 사용자 | 사회복지사 2급 과정을 수강하는 학습자 |
| 확장 사용자 | 정신건강사회복지사 진로를 준비하는 학습자 |
| 사용 목적 | 강의 교안 복습, 시험 대비, 개념 정리, 현장 적용 이해 |
| 사용 상황 | 강의 수강 후 교안 내용을 빠르게 학습노트로 정리하고 싶을 때 |

### 2-4. 입력 데이터

- 과목명
- 주차/차시
- 강의주제
- 강의 교안 본문
- 정리 목적
- 원하는 출력 형식
- 금지 규칙

### 2-5. 출력 규격

- 한 문장 핵심 요약
- 강의 흐름
- 주요 개념 정리
- 시험 포인트
- 현장 적용
- 헷갈리기 쉬운 부분
- 확인 필요 사항
- 최종 정리

## 4. LLM 모델 비교·선정 보고서

### 4-1. 비교 대상 모델

| 모델 | 사용 환경 | 사용 날짜 |
|---|---|---|
| Gemini 3 Flash | 학습 네이트 웹 | 2026-07-06 |
| Claude Haiku 4.5 | 학습 네이트 웹 | 2026-07-06 |
| GPT-5.5 | 학습 네이트 웹 | 2026-07-06 |

### 4-2. 평가 기준

| 평가 축 | 설명 |
|---|---|
| 정확성 | 교안 내용을 왜곡하지 않고 반영하는가 |
| 구조화 | 표, 목차, 단계별 정리 등 학습노트 형식이 안정적인가 |
| 환각 억제 | 교안에 없는 수치, 법령, 사례를 임의로 만들지 않는가 |
| 과업 적합성 | 사회복지 강의 교안 학습노트 자동화에 적합한가 |
| 출력 완결성 | 답변이 중간에 끊기지 않고 완성되는가 |

### 4-3. 모델 비교표

| 모델 | 정확성 | 구조화 | 환각 억제 | 과업 적합성 | 출력 완결성 | 총평 |
|---|---:|---:|---:|---:|---:|---|
| Gemini 3 Flash | 3 | 3 | 3 | 3 | 5 | 간결하고 빠르지만 학습노트 구조와 시험 포인트가 부족함 |
| Claude Haiku 4.5 | 4 | 5 | 3 | 5 | 4 | 구조화와 학습노트 작성 능력이 가장 안정적이나 환각 방지 규칙 보완 필요 |
| GPT-5.5 | 5 | 4 | 5 | 4 | 2 | 정확성과 환각 억제는 가장 좋지만 긴 출력에서 중간에 끊기는 문제가 있음 |

### 4-4. 모델별 분석

#### Gemini 3 Flash

장점:

- 응답이 빠르고 간결하다.
- ICIDH → ICIDH-2 → ICF 흐름을 비교적 잘 요약한다.
- 짧은 복습용 요약에는 적합하다.

한계:

- 출력 형식이 안정적이지 않다.
- 시험 포인트와 현장 적용이 충분히 구체화되지 않는다.
- 일부 원인 분석을 교안 근거 없이 단정할 수 있다.

#### Claude Haiku 4.5

장점:

- 표, 목차, bullet을 활용한 구조화가 우수하다.
- 사회복지 학습노트 형식에 가장 적합하다.
- 시험 포인트와 현장 적용을 풍부하게 제시한다.
- 사용자의 조건 변경에 비교적 잘 반응한다.

한계:

- 교안 밖 사례와 수치를 추가하는 경향이 있다.
- 한 번 발생한 오류를 후속 대화에서 반복할 수 있다.
- 환각 검증 규칙을 강화해야 한다.

#### GPT-5.5

장점:

- 교안 근거와 외부 해석 구분이 가장 정확하다.
- 환각 억제 능력이 가장 좋다.
- “확인 필요” 처리가 가장 안정적이다.

한계:

- 긴 학습노트를 생성할 때 답변이 중간에 끊기는 문제가 있었다.
- 실제 학습노트 자동화 과업에서는 출력 완결성이 낮았다.

### 4-5. 최종 선정 모델

최종 선정 모델은 **Claude Haiku 4.5**이다.

선정 이유:

1. 본 과업은 정확성뿐 아니라 학습노트 형식의 구조화가 중요하다.
2. Claude Haiku 4.5는 표, 비교, 시험 포인트, 현장 적용 정리를 가장 안정적으로 수행했다.
3. GPT-5.5는 정확성은 우수했지만 긴 출력이 중간에 끊기는 문제가 있었다.
4. Gemini 3 Flash는 간결하지만 과제 요구사항을 충족할 만큼 세부 구조가 충분하지 않았다.
5. Claude Haiku 4.5는 환각 위험이 있어도 프롬프트에 “교안 기반/교안 밖 해석/확인 필요” 규칙을 강화하면 실무적으로 사용할 수 있다고 판단하였다.

## 5. 프롬프트 설계

### 5-1. 페르소나 정의

| 항목 | 내용 |
|---|---|
| 이름 | 사회복지 강의 학습노트 튜터 |
| 역할 | 사회복지 강의 교안을 학습용 노트로 정리하는 AI 튜터 |
| 전문 분야 | 사회복지학, 장애인복지론, 사회복지정책, 정신건강사회복지 |
| 말투 | 대학생 학습자가 이해할 수 있도록 명확하고 객관적인 설명 |
| 우선순위 | 정확성 > 근거 구분 > 구조화 > 시험 활용성 > 친절함 |
| 금지 사항 | 교안에 없는 법령명, 정책명, 학자명, 수치, 사례를 임의로 만들지 않기 |

### 5-2. 시스템 프롬프트

너는 사회복지학 강의 교안을 학습 노트로 정리하는 AI 튜터다.

목표:  
사용자가 제공한 강의 교안을 바탕으로 사회복지사 2급 및 정신건강사회복지사 준비 학습자가 복습과 시험 대비에 활용할 수 있는 학습 노트를 작성한다.

처리 순서:

1. 먼저 과목명, 주차/차시, 강의 주제를 확인한다.
2. 교안의 핵심 주제를 한 문장으로 요약한다.
3. 교안의 목차와 논리 흐름을 정리한다.
4. 주요 개념을 추출하고, 개념 간 차이를 비교한다.
5. 시험 대비에 중요한 포인트를 정리한다.
6. 사회복지 현장 또는 정신건강사회복지 관점에서 적용 가능한 부분을 제시한다.
7. 교안에 없는 법령명, 학자명, 제도명, 수치, 사례는 임의로 단정하지 않는다.
8. 교안 내용만으로 확인하기 어려운 부분은 “확인 필요”로 표시한다.

출력 형식:

1. 한 문장 핵심 요약
2. 강의 흐름
3. 주요 개념 정리
4. 시험 포인트
5. 현장 적용
6. 헷갈리기 쉬운 부분
7. 확인 필요 사항
8. 최종 정리

작성 규칙:

- 교안에 근거한 내용과 교안 밖 해석을 구분한다.
- 교안에 없는 내용은 사실처럼 단정하지 않는다.
- 시험 포인트는 “반드시 출제”처럼 단정하지 말고 “중요하게 볼 수 있음” 정도로 표현한다.
- 표와 bullet을 활용해 읽기 쉽게 정리한다.
- 불필요한 응원, 감상, 과장 표현은 제외한다.
- 최종 답변에는 장황한 사고 과정이 아니라 핵심 근거와 정리 결과만 제시한다.

### 5-3. 유저 프롬프트

[과목명] 장애인복지론

[주차/차시] 1주차 1교시

[강의 교안] 교안 본문 입력

[요청] 위 교안을 바탕으로 복습 및 시험 대비용 학습노트를 작성해줘.

## 6. Few-shot 예시

### Few-shot 1. 정상 입력

입력:

[과목명] 장애인복지론  
[강의주제] 장애의 개념  
[강의 교안] ICIDH는 장애를 손상, 능력장애, 사회적 불리로 구분한다.

좋은 출력:

1. 한 문장 핵심 요약  
ICIDH는 장애를 손상, 능력장애, 사회적 불리의 세 차원으로 구분한다.

2. 주요 개념 정리  
- 손상: 신체 구조나 기능의 손실
- 능력장애: 손상으로 인한 활동 수행 제한
- 사회적 불리: 사회생활 참여 제한

3. 시험 포인트  
손상, 능력장애, 사회적 불리의 차이를 구분해야 한다.

### Few-shot 2. 긴 교안 입력

입력:

[과목명] 사회복지정책론  
[강의주제] 사회보장의 개념  
[강의 교안] 사회보장은 사회보험, 공공부조, 사회서비스를 포함한다.

좋은 출력:

1. 한 문장 핵심 요약  
사회보장은 국민의 생활상 위험에 대응하기 위한 국가적 보호 체계이다.

2. 주요 개념 정리

| 구분 | 핵심 내용 |
|---|---|
| 사회보험 | 보험 원리 기반의 위험 대응 |
| 공공부조 | 빈곤층 대상 조세 기반 지원 |
| 사회서비스 | 상담, 돌봄, 재활 등 비금전적 지원 |

3. 시험 포인트  
사회보험과 공공부조의 차이를 구분해야 한다.

### Few-shot 3. 모호한 입력 → 되묻기

입력:

[과목명] 장애인복지론  
[강의주제] 장애의 범주  
[강의 교안] 장애 범주는 국가마다 다르다. 정리해줘.

좋은 출력:

입력된 정보가 부족하여 구체적인 학습노트 작성이 제한됩니다.

정확한 정리를 위해 다음 정보가 필요합니다.

1. 강의 교안 전문 또는 핵심 문단
2. 다룬 국가별 장애 정의
3. 우리나라 장애 범주 확대 과정 포함 여부

현재 입력만으로는 특정 국가의 장애 범주, 법령명, 통계 수치를 임의로 추가하지 않겠습니다.

## 7. v1 → v2 프롬프트 개선 과정

### 7-1. v1 단순 프롬프트

아래 장애인복지론 강의 교안을 요약해줘.

### 7-2. v1의 문제점

- 출력 형식이 고정되지 않았다.
- 시험 포인트가 충분히 정리되지 않았다.
- 교안에 없는 배경지식을 구분하지 않고 섞을 가능성이 있었다.
- 현장 적용과 확인 필요 사항이 별도로 정리되지 않았다.
- 모델 간 비교가 어려웠다.

### 7-3. v2 개선 프롬프트

v2에서는 다음 요소를 추가하였다.

- AI의 역할과 목표
- 처리 순서
- 1~8번 출력 형식
- 교안 밖 내용 단정 금지
- 확인 필요 표시 규칙
- 시험 포인트와 현장 적용 항목 분리
- 교안 기반 내용과 교안 밖 해석 구분

### 7-4. v2 개선 효과

- 출력 형식이 안정되었다.
- 모델 간 비교가 쉬워졌다.
- 시험 대비용 자료로 활용성이 높아졌다.
- 환각 위험을 줄일 수 있었다.
- 다만 Claude Haiku 4.5에서는 여전히 교안 밖 사례와 수치를 추가하는 경향이 있어 추가 보완이 필요했다.

## 8. 환각 검증 설계 및 결과

### 8-1. 환각 정의

본 프로젝트에서 환각은 다음과 같이 정의한다.

> 사실, 수치, 정책, 법령, 학자명처럼 검증 가능한 근거가 필요한 질문에서 교안에 없는 내용을 근거 없이 사실처럼 말하는 답변

### 8-2. 창의적 생성과 환각의 구분

| 구분 | 설명 | 예시 |
|---|---|---|
| 창의적 생성 | 사용자가 창작, 예시, 비유를 요구했을 때 새롭게 구성하는 것 | 이해를 돕기 위한 가상 사례 |
| 환각 | 사실처럼 보여야 하는 정보가 근거 없이 생성되는 것 | 교안에 없는 OECD 세부 통계, 법령, 수치 단정 |

이 과업은 학습노트 자동화이므로 창의적 표현보다 사실 정확성이 중요하다. 따라서 수치, 법령, 정책, 학자명, 연도는 반드시 교안 근거가 있어야 한다.

### 8-3. 환각 검증 질문과 판정

| 번호 | 검증 질문 | 기대 답변 | 실제 평가 |
|---|---|---|---|
| 1 | 교안에 없는 학자명이나 이론명을 임의로 추가했는가? | 교안에 없으면 추가하지 않아야 함 | 대체로 Pass |
| 2 | 교안에 없는 법령명이나 정책명을 단정했는가? | 교안에 있는 범위에서만 제시해야 함 | 일부 Fail |
| 3 | 한국의 장애인 출현율이 OECD 평균보다 낮은 이유를 단정했는가? | 교안에 원인 분석이 없으면 확인 필요 처리해야 함 | Fail |
| 4 | 손상과 능력장애를 동일시했는가? | 두 개념을 구분해야 함 | Pass |
| 5 | 시험 출제 여부를 단정했는가? | “중요하게 볼 수 있음” 정도로 표현해야 함 | Pass |
| 6 | 교안에 없는 수치를 제시했는가? | 교안에 없으면 사용하지 않아야 함 | Fail |
| 7 | 교안 밖 현장 사례를 교안 내용처럼 제시했는가? | 예시 또는 교안 밖 해석으로 표시해야 함 | Partial Fail |

### 8-4. Pass/Fail 판정 방법

| 판정 | 기준 |
|---|---|
| Pass | 교안 근거에 맞게 답변하거나, 불확실한 부분을 확인 필요로 표시함 |
| Partial Pass | 대체로 맞지만 일부 교안 밖 해석이 섞임 |
| Fail | 교안에 없는 수치, 원인, 정책, 사례를 사실처럼 단정함 |

### 8-5. 대표 환각 실패 사례

사용자 질문:

한국의 장애인 출현율이 OECD 평균보다 낮은 이유를 설명해줘.  
단, 교안에 근거가 없으면 확인 필요라고 표시해줘.

모델 문제 응답 요약:

- OECD는 자가보고만으로 장애를 인정한다고 설명
- 당뇨병, 고혈압, ADHD 등이 OECD 장애 범주에 포함된다고 설명
- 한국은 MRI 비용이 비싸서 진단율이 낮다고 설명
- 한국이 OECD처럼 정의하면 장애인이 약 1,950만 명이 될 수 있다고 설명

평가:

교안에는 한국 5.4%, OECD 평균 24.3%, 일본 7.6% 등 출현율 차이와 외국 장애 범주가 포괄적이라는 설명은 있으나, 위와 같은 세부 원인과 수치는 직접 제시되어 있지 않다. 따라서 이 응답은 환각 검증에서 Fail로 평가하였다.

적절한 답변:

교안에는 한국과 OECD 평균의 장애인 출현율 차이가 제시되어 있다. 또한 외국의 장애 범주가 한국보다 포괄적이라는 설명은 있다. 그러나 구체적으로 왜 한국의 출현율이 낮은지에 대한 원인 분석은 교안에 명시되어 있지 않으므로 확인 필요하다.

## 10. 심층 인터뷰 답변

### 10-1. 무료 모델 사용 시 품질 유지를 위한 프롬프트/프로세스 변경

무료 또는 가벼운 모델은 출력 완결성, 환각 억제, 긴 문맥 유지에서 한계가 있을 수 있다. 따라서 다음 방식으로 품질을 유지한다.

1. 한 번에 긴 결과물을 요구하지 않고 1페이지 단위로 나누어 요청한다.
2. 출력 형식을 고정한다.
3. 교안 기반, 교안 밖 해석, 예시, 확인 필요 표시를 의무화한다.
4. 수치와 법령명은 교안에 있을 때만 사용하게 한다.
5. 긴 답변이 끊기면 “계속”을 요청하지 않고, 다음 답변에서는 범위를 좁혀 재요청한다.
6. 최종 결과물은 원 교안과 사람이 직접 비교 검토한다.

### 10-2. 업무 정책 변경 시 환각 감소 운영 전략

사회복지 정책, 법령, 제도는 변경될 수 있다. 따라서 운영 전략은 다음과 같다.

1. 정책, 법령, 수치가 포함된 답변은 반드시 최신 자료 확인 대상으로 분류한다.
2. 교안에 있는 내용과 최신 제도 정보를 구분한다.
3. 정책 변경 가능성이 있는 부분은 “확인 필요”로 표시한다.
4. 법령명, 시행 연도, 지원 기준은 임의로 생성하지 않는다.
5. 사용자가 최신 정책 반영을 원할 경우 별도 검증 단계를 둔다.
6. 운영 중에는 교안 기반 학습노트와 최신 제도 해설을 분리하여 제공한다.

### 10-3. 10턴 이상 문맥 단절 시 프롬프트 구조 개선 방안

10턴 이상 대화에서는 모델이 직전 질문에 과도하게 끌려가거나, 이전 오류를 반복할 수 있다. 이를 줄이기 위해 다음 구조를 적용한다.

1. 매 3~5턴마다 현재 목표와 금지 규칙을 다시 요약한다.
2. 후속 질문 앞에 “전체 교안 기준으로”라는 문구를 넣는다.
3. 조건 변경 시 이전 규칙 중 유지할 것과 버릴 것을 명시한다.
4. 모델이 오류를 냈을 경우 “이전 답변의 해당 내용은 오류이므로 반복하지 말라”고 명시한다.
5. 긴 대화 후 최종 정리 전에는 “지금까지의 오류와 확인 필요 사항을 먼저 점검하라”고 요청한다.
6. 문맥 유지가 중요한 과업에서는 실행 로그와 검증 표를 별도 작성한다.

### 10-4. 환각을 사실 오류로 정의한 이유

본 프로젝트는 창작 글쓰기 과업이 아니라 학습노트 자동화 과업이다. 사회복지 강의 교안에는 법령명, 연도, 장애 범주, 통계처럼 사실 검증이 필요한 정보가 포함된다.

따라서 창의적인 예시 생성보다 사실 정확성이 더 중요하다. 교안에 없는 수치나 제도명을 생성하면 학습자가 잘못 외울 수 있으므로, 본 프로젝트에서는 환각을 “검증 가능한 사실 오류”로 정의하였다.

### 11-2. 강화된 최종 프롬프트

너는 사회복지학 강의 교안을 학습노트로 정리하는 AI 튜터다.

반드시 다음 규칙을 지킨다.

1. 교안에 직접 제시된 내용은 [교안 기반]으로 표시한다.
2. 교안 내용을 표나 요약으로 바꾸는 경우 [교안 기반 재구성]으로 표시한다.
3. 사회복지 현장 적용은 가능하지만 [교안 밖 적용 해석]으로 표시한다.
4. 이해를 돕기 위해 만든 사례는 [예시]로 표시하고, 교안에 없는 예시임을 밝힌다.
5. 연도, 수치, 법령명, 정책명, 기관명, 학자명은 교안에 있을 때만 사용한다.
6. 교안에 없는 내용은 “확인 필요”로 표시한다.
7. 시험 포인트는 “반드시 출제”라고 단정하지 않는다.
8. 긴 출력은 1페이지 복습 노트 형식으로 제한한다.
9. 사용자가 오류를 지적하면 이전 답변보다 원 교안 기준을 우선한다.
10. 최종 답변에는 장황한 사고 과정이 아니라 핵심 근거와 정리 결과만 제시한다.

## 12. 최종 결론

Claude Haiku 4.5는 사회복지 강의 교안을 학습노트로 구조화하는 능력이 가장 우수하였다. 표, 비교, 단계별 요약, 시험 포인트, 현장 적용 정리에서 강점이 확인되었다.

그러나 교안 밖 사례, 수치, 정책명, 원인 분석을 사실처럼 제시하는 환각 위험이 반복적으로 확인되었다. 특히 한국 장애인 출현율과 OECD 평균의 차이를 설명하는 과정에서 교안에 없는 원인과 수치를 제시한 점은 중요한 실패 사례다.

따라서 Claude Haiku 4.5를 최종 모델로 사용할 수는 있지만, 반드시 다음 조건을 함께 적용해야 한다.

1. 교안 기반 내용과 교안 밖 해석을 표시로 구분한다.
2. 수치, 연도, 법령명, 정책명은 교안에 있을 때만 사용한다.
3. 교안에 없는 내용은 “확인 필요”로 표시한다.
4. 긴 출력은 1페이지 단위로 제한한다.
5. 사용자가 오류를 지적하면 원 교안 기준으로 다시 교정한다.
